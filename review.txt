**score understanding → phrase segmentation → fingering optimization**.

Tôi sẽ review theo 5 góc:

1️⃣ Architecture correctness
2️⃣ Music cognition realism
3️⃣ Algorithmic soundness
4️⃣ Production risk
5️⃣ Những cải tiến đáng giá nhất

# 2. IO Layer review

### Điểm tốt

Bạn parse:

* grand staff
* `<backup>` time reset
* slur
* articulation
* fingering GT
* key signature

→ **đủ cho 95% sheet piano thực tế**

### Tuy nhiên còn thiếu 3 thứ quan trọng

#### (1) Beam grouping

Beam giúp nhận diện **gesture**.

Ví dụ:

```
♪♪♪♪   ♪♪♪♪
```

Beam thường = **1 motor gesture**

Bạn nên parse:

```
<beam>
```

để hỗ trợ:

* phrase micro segmentation
* fingering grouping.

---

#### (2) Tie

Tie ≠ slur.

Tie ảnh hưởng:

* fingering substitution
* legato modelling.

Cần parse:

```
<tie type="start/stop">
```

---

#### (3) Chord flag

MusicXML:

```
<chord/>
```

Giúp phân biệt:

```
simultaneous onset
```

vs

```
polyphonic voices
```

Nếu không parse kỹ → DP sẽ bị nhiễu.

# 3. Recognition Layers review

## Layer 0 — ScoreProfile

Rất đúng hướng.

Bạn có thể thêm:

```
tempo
meter type
texture density
```

Ví dụ:

```
3/4 waltz
6/8 compound
```

vì phrase prior khác nhau.

---

## Layer 1 — Voice Separation

Bạn dùng:

```
highest note per onset = melody
```

Đây là heuristic tốt cho:

* homophonic texture

Nhưng sẽ fail với:

* Chopin
* Debussy
* Bach

Ví dụ:

melody nằm **giữa chord**.

Giải pháp tốt hơn:

```
skyline algorithm
+ duration weighting
```

hoặc

```
probabilistic melody extraction
```

---

# 4. Phrase Boundary Detector

Đây là phần **rất tốt**.

Bạn dùng:

```
Logit + sigmoid
```

→ tốt hơn threshold.

Signals bạn dùng cũng hợp lý:

* slur
* rest
* cadence
* agogic
* melodic arc

Tuy nhiên tôi khuyên thêm 2 signal cực mạnh.

---

## Signal 1: Metric boundary

Phrase thường kết ở:

```
strong beat
measure end
```

Feature:

```
beat_position
```

---

## Signal 2: Rhythmic lengthening

Trước phrase end thường:

```
note duration ↑
```

gọi là:

**phrase-final lengthening**

---

# 5. Global Segmentation (Viterbi)

Đây là phần **rất đúng hướng**.

Objective:

```
max Σ P(boundary) × Prior(length)
```

Rất hợp lý.

Nhưng Gaussian prior:

```
μ=4
σ=2
```

không đúng cho mọi genre.

Ví dụ:

| style     | phrase |
| --------- | ------ |
| Baroque   | 2–4    |
| Classical | 4      |
| Romantic  | 4–8    |
| Pop       | 4      |

Giải pháp:

```
style adaptive prior
```

# 7. Fingering DP review

Đây là phần **xuất sắc nhất của kiến trúc**.

Bạn đã encode:

### biomechanics

* tendon coupling
* span
* tempo
* black key depth

Điều này rất gần với research paper:

```
Piano Fingering Modeling via Biomechanical Constraints
```

---

### Thumb-under reward

Ý tưởng đúng.

Nhưng reward **không nên fixed**.

Better:

```
reward = f(interval, direction)
```

Ví dụ:

```
3 → 1 scale crossing
```

reward lớn.

---

### Finger 4 penalty

Đúng về pedagogy.

Nhưng cần soft:

```
penalty ≈ 0.2
```

vì nhiều passage **bắt buộc dùng 4**.

---

# 8. PhraseIntentAnalyzer

Ý tưởng:

```
cantabile
brilliant
expressive
```

Rất hay.

Nhưng detection sẽ **khó chính xác** nếu chỉ dựa score.

Tôi khuyên:

intent nên derive từ:

```
tempo
articulation
density
```

Ví dụ:

```
many staccato + fast tempo
→ brilliant
```

# 9. CrossPhraseStitch

Đây là nơi **hầu hết hệ thống fingering fail**.

Bạn đã có:

```
junction constraint
```

Rất tốt.

Nhưng cần thêm:

```
lookahead = 3–5 notes
```

để tránh:

```
phrase1 end finger
phrase2 start finger
```

bị conflict.


Bạn có thể thêm:

### Rule ví dụ

```
span > max_span
```

```
4-5 alternation fast tempo
```

# 11. Điểm yếu lớn nhất của kiến trúc

Có 2 điểm.

---

## Weakness 1 — Chord fingering

Hiện tại:

```
RH [1,3,5]
LH [5,3,1]
```

chỉ đúng với **triad root position**.

Bạn cần detect:

```
inversion
7th chord
cluster
```

---

## Weakness 2 — Pattern detection chưa đủ mạnh

Pattern nên detect **trước DP**.

Ví dụ:

```
scale run 12 notes
```

DP không nên xử lý từng note.

→ áp dụng template.
